# Шаг 13. Yolo shortcut — запуск Claude без подтверждений

Что даёт: алиас `yolo` (или своё имя) запускает Claude с флагом `--dangerously-skip-permissions` — Claude перестаёт спрашивать разрешение на каждое действие (запуск команд, чтение файлов, правки). Сильно ускоряет работу, но **снимает защиту**.

> ⚠️ **Важно понимать:** этот режим даёт Claude полную свободу выполнять команды на твоём ноуте без твоего разрешения. Используй только когда:
> 1. Ты доверяешь задаче, которую дала Claude
> 2. Ты понимаешь, что он может сделать (изменить файлы, запустить команды, отправить сетевые запросы)
> 3. Ты не работаешь с production / критичными данными
>
> Для повседневных задач франчайзи (посты, ответы родителям, расписание) это **безопасно**. Для команд "удали все файлы в этой папке" / "разверни новый сервер" — лучше **не** использовать yolo, а контролировать каждое действие.

## Когда стоит включить

✅ Хорошо использовать yolo:
- Пишешь пост или ответ родителю — Claude читает шаблоны, обращается к скиллам, выдаёт готовый текст
- Парсишь Google Sheets, делаешь сводку
- Ищешь информацию через Exa / читаешь сайты через Chromium MCP
- Работаешь со своими markdown-заметками

❌ Лучше **не** использовать yolo (запускай обычный `claude`):
- Удаление файлов / папок
- Установка нового софта системного уровня
- Работа с git push в рабочих репозиториях
- Команды на VPS / удалённых серверах
- Когда впервые используешь незнакомый скилл и не знаешь что он делает

Можно держать **оба варианта** — `claude` для осторожной работы, `yolo` для быстрой.

## Установка

### Mac (zsh — стандарт macOS 13+)

Открой `~/.zshrc`:
```bash
nano ~/.zshrc
```

В конец файла добавь:
```bash
alias yolo='claude --dangerously-skip-permissions'
```

Сохрани (Ctrl+O → Enter → Ctrl+X). Перечитай:
```bash
source ~/.zshrc
```

Проверь:
```bash
yolo --version
```

Должен ответить версией Claude Code.

### Windows — PowerShell (рекомендуется)

Самый чистый способ. Открой профиль PowerShell:
```powershell
notepad $PROFILE
```

Если файла нет — согласись создать. В нём добавь:
```powershell
function yolo { claude --dangerously-skip-permissions @args }
```

Сохрани, закрой Notepad, **перезапусти PowerShell**. Теперь:
```powershell
yolo
```

Запустит Claude в режиме без подтверждений. С аргументом:
```powershell
yolo .                              # текущая папка
yolo C:\path\to\project             # конкретная папка
```

### Windows — bat-файл (для cmd или двойного клика)

Создай файл `yolo.bat` в любой папке, прописанной в PATH (например, `%USERPROFILE%\bin\` — мы его создали в Шаге 2):

```bat
@echo off
claude --dangerously-skip-permissions %*
```

Теперь из cmd или PowerShell — `yolo` работает.

### Windows — ярлык с хоткеем (как macOS Shortcuts)

Если хочется глобальный хоткей:

1. На рабочем столе: ПКМ → **Создать** → **Ярлык**
2. В поле "объект" укажи:
   ```
   C:\Windows\System32\cmd.exe /k claude --dangerously-skip-permissions
   ```
3. Имя ярлыка — `Yolo Claude` или как нравится
4. После создания: ПКМ по ярлыку → **Свойства** → поле **"Быстрый вызов"** → нажми сочетание клавиш (например, **Ctrl+Alt+C**)
5. Ярлык должен лежать на рабочем столе **или** в Start Menu — иначе хоткей не сработает

Теперь Ctrl+Alt+C из любого окна — открывает Claude в yolo-режиме.

### Git Bash / WSL — как на Mac

В `~/.bashrc`:
```bash
alias yolo='claude --dangerously-skip-permissions'
```

Перечитай: `source ~/.bashrc`.

## Что ещё можно сделать в alias / function

Добавь дополнительные дефолты — например, всегда нужная папка:

**Mac/Linux:**
```bash
alias yolo='cd ~/altairika && claude --dangerously-skip-permissions'
```

**PowerShell:**
```powershell
function yolo {
    Set-Location $env:USERPROFILE\altairika
    claude --dangerously-skip-permissions @args
}
```

Имя `yolo` можно сменить на любое — `cy`, `claudego`, `гойда` (если транслит-имя ОК для shell).

## Если что-то пошло не так

- **Mac:** `alias` пропал после перезагрузки → проверь, что `.zshrc` сохранился, и shell — именно zsh (`echo $SHELL`)
- **Windows PowerShell:** `function` не появилась → запусти `$PROFILE` в PowerShell, проверь путь — иногда Notepad сохраняет в другую папку
- **Команда `claude` не находится в новом терминале** → PATH не подхватился, перезапусти компьютер ИЛИ проверь Шаг 2 (добавление `~/.local/bin` в PATH)
