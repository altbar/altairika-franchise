# Шаг 12. Cmux session restore (опционально, только Mac)

> Этот шаг — для технически продвинутых партнёров, кто работает в терминале часто и держит несколько Claude-сессий параллельно. Если ты не понимаешь, зачем тебе несколько одновременных Claude — пропускай этот шаг.

## Что даёт

Cmux — терминальный менеджер сессий (форк tmux с GUI-управлением). Позволяет:
- Держать несколько Claude-сессий одновременно (например, одна про SMM, одна про родителей, одна про админку)
- Не терять состояние Claude при закрытии окна терминала
- Автоматически восстанавливать сессии при перезапуске Mac

Альтернатива: tmux (классический), Zellij, или просто несколько вкладок iTerm2.

## Установка cmux

```bash
brew install --cask cmux
```

Или скачай с https://cmux.ai → перенеси в Applications.

При первом запуске:
- Дай разрешение на Accessibility (System Settings → Privacy → Accessibility)
- Согласись на установку shell-integration

## Базовое использование

Запусти cmux. Откроется GUI с панелями.
- **Cmd + T** — новая сессия (новая панель/таб)
- **Cmd + W** — закрыть текущую
- **Cmd + 1, 2, 3** — переключение между сессиями

В каждой сессии запускай свою задачу:
- Сессия 1: `cd ~/altairika-franchise && claude` — работа с китом
- Сессия 2: `cd ~/SMM && claude` — посты в соцсети
- Сессия 3: `cd ~/Bookings && claude` — таблицы броней

## Восстановление сессий после перезапуска Mac

Cmux может автоматически перезапускать Claude-сессии после ребута. Это требует настройки.

### Шаг 1 — установи скрипт автовосстановления

В файл `~/.zshrc` добавь:
```bash
# Auto-restore Claude sessions on cmux startup
if [ -n "$CMUX_SESSION_NAME" ] && [ -f ~/.claude/sessions/$CMUX_SESSION_NAME ]; then
    last_dir=$(cat ~/.claude/sessions/$CMUX_SESSION_NAME)
    cd "$last_dir" 2>/dev/null
    claude --resume 2>/dev/null
fi
```

### Шаг 2 — сохраняй сессии при выходе

В `~/.zshrc` ниже добавь:
```bash
# Save Claude session marker on cmux pane close
if [ -n "$CMUX_SESSION_NAME" ]; then
    mkdir -p ~/.claude/sessions
    echo "$PWD" > ~/.claude/sessions/$CMUX_SESSION_NAME
fi
```

После — `source ~/.zshrc` и перезапусти cmux.

> Это упрощённая схема — для полной интеграции с inode-блокировкой cmux-сокета и т.п. см. документацию cmux. Большинству франчайзи достаточно базового запуска без auto-restore.

## Когда это НЕ нужно

- Работаешь с Claude эпизодически, по одной задаче за раз
- Не любишь горячие клавиши и многооконность
- Windows — cmux на Windows не работает (это Mac-only)
